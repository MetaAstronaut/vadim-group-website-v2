# DESIGN SYSTEM v2 — The Vadim Group

Brand: Rapid-response, high-skill repair & restoration  
UX Tone: Expert & Caring (60%) + Calm Premium Confidence (40%)  
Visual Style: Warm, minimal, premium, без “кричащего” маркетинга

---

## 1. Core Principles

1. **Trust first** — дизайн должен в первую очередь вызывать доверие, а не удивление.
2. **Calm urgency** — ощущение, что проблему решат быстро, но без паники.
3. **Clarity over decoration** — минимум визуального шума, максимум читабельности.
4. **Human-centered** — фокус на человеке и результате, а не на “сервисе как продукте”.
5. **Consistency** — одинаковые паттерны для кнопок, карточек, отступов, заголовков.

---

## 2. Color System

> Все цвета использовать как **tokens**, не как “жёсткие значения в компонентах”.

### 2.1. Brand Colors

- `--color-bg-main`: `#070B12` – тёмный фон (главный фон сайта, hero, крупные секции)
- `--color-bg-elevated`: `#0C1A27` – приподнятые блоки, карточки на тёмном фоне
- `--color-primary`: `#0C1A27` – цвет основной кнопки и ключевых акцентов
- `--color-primary-hover`: `#0A1520`
- `--color-accent-warm`: `#C6A778` – мягкий “brass / aged gold” акцент (иконки, линии, детали)
- `--color-border-subtle`: `#1C2630`
- `--color-neutral-100`: `#F5F7FA` – основной светлый текст/фон
- `--color-neutral-200`: `#E0E4EA`
- `--color-neutral-600`: `#7B8794`
- `--color-danger`: `#C44B4B` (аккуратный, при необходимости)

### 2.2. Usage Rules

- Фон сайта — тёмный (`bg-main`) с блоками `bg-elevated`.
- Акцентный “золото-бежевый” (`accent-warm`) — **только для тонких элементов**:
  - линии разделителей
  - иконки outline
  - мелкие заголовки/подзаголовки
- Не красить в акцент большие плашки и кнопки, чтобы не выглядело дешево.

## 3. Typography

### 3.1. Font Pairing (целевое)

- **Headings (H1–H3):** современный serif или high-contrast sans (например, Playfair Display / DM Serif / или аналогичная)
- **Body / UI текст:** нейтральный современный sans (например, Inter / IBM Plex Sans / SF UI)

_(Фактический выбор шрифта можно уточнить позже, но важно соблюсти pairing: expressive headings + readable body)_

### 3.2. Иерархия

- `H1` — 40–48px, жирный, tight line-height, используется 1 раз на страницу.
- `H2` — 28–32px, для ключевых секций.
- `H3` — 20–24px, подзаголовки внутри секций.
- `Body` — 16–18px, line-height 1.6–1.8 для читабельности.
- `Small` — 13–14px для подсказок, подпунктов.

### 3.3. Стиль текста

- Короткие предложения.
- Много списков (`•`, `-`, нумерация).
- Не использовать ALL CAPS, кроме маленьких акцентов (например, “EMERGENCY SERVICE”).

## 4. Layout & Spacing

### 4.1. Grid

- Основная ширина контейнера: `max-width: 1100–1200px`
- Отступы по бокам: `16px` на мобильных, `24–32px` на десктопе

### 4.2. Vertical Rhythm

Использовать модульные отступы, например:

- Между большими секциями: `80–96px`
- Между заголовком и текстом: `16–24px`
- Между карточками/элементами: `24–32px`

### 4.3. Corners (твой выбор A)

- Радиус углов: `border-radius: 6px` для:
  - кнопок
  - карточек
  - инпутов
- Крупные блоки (hero background, большие панели) можно делать квадратными или со скруглением `8px`, но **без “таблеток” и чрезмерной округлости**.

## 5. Components

### 5.1. Buttons

**Primary Button (Request Quote, main CTA)**

- background: `var(--color-primary)` (`#0C1A27`)
- color: `var(--color-neutral-100)`
- border-radius: `6px`
- padding: `14px 28px`
- font-weight: `600`
- transition: `all 0.22s ease-out`

States:

- `:hover` → `background: var(--color-primary-hover)`, лёгкая тень
- `:active` → немного темнее + меньше тени
- `:focus-visible` → outline `2px`, цвет `accent-warm`
- `:disabled` → `opacity: 0.5`, курсор `not-allowed`

**Secondary Buttons / Links**

- Текстовые кнопки без плашки
- Цвет текста: `neutral-100` или `accent-warm`
- Подчёркивание при `hover`

---

### 5.2. Cards (Services, Testimonials)

- Фон: `bg-elevated` или `rgba(255,255,255,0.02)` на тёмном фоне
- Бордер: `1px solid var(--color-border-subtle)`
- Тени: мягкие, слегка размытые, без ярких “всплытий”
- Внутри:
  - Заголовок H3
  - Краткий абзац (max 3–4 строки)
  - List bullets или key points

Допустимо использовать subtle hover:

- лёгкое поднятие на 2–3px
- увеличение яркости бордера

---

### 5.3. Sections

**Hero Section:**

- Тёмный фон, минимум отвлекающих деталей.
- Левая колонка: заголовок + подзаголовок + 1–2 bullets + кнопка.
- Правая колонка:
  - либо фото/фон,
  - либо “before/after”-визуал,
  - либо “Vadim + short intro”.

**Testimonials:**

- Карточки на чуть более светлом фоне.
- Имя клиента + район (e.g., “Lake Nona, Orlando”)
- Текст в 2–4 строки, без огромных полотен.

**Process:**

- Линейный или шаговый формат: 1 → 2 → 3 → …
- Каждый шаг — карточка/иконка с кратким описанием.

---

### 5.4. Forms

- Очень простые
- Визуально лёгкие, без лишних рамок
- Каждый input:
  - `border-radius: 6px`
  - `border: 1px solid var(--color-border-subtle)`
  - `background: #050910` или близкий

Микрокопирайт:

- Под кнопкой обязательно краткое сообщение типа:
  - “We’ll reply within 2–4 business hours.”
  - “No spam. No pressure.”

## 6. Imagery

### 6.1. Стиль фото

- Clean before/after editorials (ты выбрал C:2)
- Чистые, без лишних людей в кадре
- Акцент на:
  - качестве поверхности
  - аккуратности
  - деталях

### 6.2. Фотографии с Vadim

- Естественные, не постановочные.
- В процессе работы (ремонт, осмотр, пояснение клиенту).
- Одежда — чистая, профессиональная (без “builder clown style”).

## 7. Motion & Interaction

- Анимации → мягкие, короткие (`180–240ms`).
- Основные эффекты:
  - fade-in при появлении блоков
  - лёгкий `translateY` вверх для карточек при hover
- Никаких:
  - bounce-анимаций
  - резких масштабирований
  - мигающих элементов

---

## 8. Accessibility

Обязательные правила:

- Контраст текста к фону не ниже WCAG AA:
  - Основной текст: контраст ≥ 4.5:1
  - Крупный текст (H1/H2): ≥ 3:1
- Все интерактивные элементы имеют:
  - `:focus-visible` состояния
  - достаточную зону клика (минимум 40×40px)
- Текст никогда не должен быть меньше 14px на мобильных.

---

## 9. Content & Messaging Hooks

- Использовать фразы:
  - “urgent situations handled calmly and professionally”
  - “repairs done right the first time”
  - “we take personal responsibility for the work”
- Не использовать:
  - CapsLock, чрезмерные восклицательные знаки
  - “24/7!!!”, “Best in Orlando!!!” и т.п.

---

## 10. Implementation Notes (для разработчиков и AI-агентов)

- Все цвета, отступы, шрифты оформить как **CSS variables / Tailwind tokens**.
- Не хардкодить значения стилей в нескольких местах.
- При создании новых компонентов:
  - использовать эти же отступы, радиусы, цвета.
  - не добавлять новые оттенки без согласования.
- Любой новый UI-элемент должен:
  - выглядеть как часть системы,
  - а не как “чужой компонент из шаблона”.
